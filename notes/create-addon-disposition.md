## Set aside for pressure-point meetings

These are interesting, but they are not simple addon adds. They touch faction resource control, travel pressure, territory, trade, or economy and should be discussed with partners before implementation.

### Create Ore Excavation

Decision: **set aside for pressure-point meeting**.

Reason: the idea is genuinely compelling if tuned correctly. Infinite/resource-vein machines could turn mining into visible outposts and faction claims. The design direction would be:

- slow or reduce ordinary infinite ore-generation machines;
- make discovered infinite veins clearly better;
- make extraction sites worth defending/trading/raiding;
- connect outposts to rails, roads, or cargo routes.

This needs group discussion because it changes resource economy and territorial conflict.

### Create: Molten Vents

Decision: **set aside for pressure-point meeting**.

Reason: like oilfields and ore veins, vents could make specific world locations valuable. This belongs with territory/resource design, not casual addon inclusion.

### Create: Trading Floor

Decision: **defer to pressure/economy meeting**.

Reason: villager/trade automation can become a major economy pressure point or a broken resource exploit. Discuss alongside markets, emeralds, XP, gear, and faction trade.

### Create: Hypertubes

Decision: **defer to pressure/travel meeting**.

Reason: could be very fun as faction transit infrastructure, but also risks bypassing roads, rails, mounts, Aeronautics, and border geography. If added later, it should probably be gated after trains and treated as physical infrastructure, not cheap teleportation.

## Deferred for separate technical/balance audits

### Create Crafts & Additions

Decision: **not added yet; audit complete; possible future electricity pass**.

Reason: electricity is unfamiliar and can change Create's feel. See `notes/create-crafts-additions-audit.md`. Tentative stance is yes only if gated/configured as precision-era infrastructure.

### Create: New Age

Decision: **unknown / not now**.

Reason: overlaps with the electricity/advanced-power question and may push Tenpack toward tech-pack territory. Revisit only after deciding whether Crafts & Additions belongs.

### Create: Pattern Schematics

Decision: **soft maybe; not immediate**.

Reason: could help faction builders repeat station/factory/hangar patterns, but it is not urgent and the value is unclear without seeing it in play. Add only if it improves building workflow without adding clutter.

## Out for now

These are not aligned with the current Tenpack Create/factions philosophy or are too likely to add noise/bypasses.

### Create Mechanical Extruder

Decision: **out**.

Reason: renewable block/material generation feels like bullshit for this pack unless specifically redesigned as a pressure-point system. It risks trivializing resource/building economies.

### Create: The Factory Must Grow

Decision: **out for now**.

Reason: too heavy. It may be interesting someday, but current Tenpack should not jump into a full heavy-industry kitchen sink before the core Create/oil/rails/flight loop is proven.

### Create: Connected

Decision: **out for now**.

Reason: too easy / too much utility convenience. Utility addons often hide progression bypasses and reduce the need for interesting machinery.

### Create: Configurable Machine Outputs

Decision: **no**.

Reason: not needed unless a specific output-control problem appears.

### Create Jetpack

Decision: **out**.

Reason: personal flight conflicts with Aeronautics progression, aircraft logistics, roads, rails, mounts, and terrain pressure.

### Create Stuff 'N Additions

Decision: **out**.

Reason: too likely to add mobility/tool bypasses and general gadget noise.

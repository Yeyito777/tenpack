# 0.7 Update Changelog

## Backend
- CrackersLib bumped to 1.20.1-0.4.6

## Additions/Tweaks
- Added transparent cubes to the edges of cloud types #2 #33 #51 #59 #69 #106
  - This is a new customizable value, ``transparency_fade``, which can be added when defining a cloud type using JSON
  - Used by stratocumulus, cumulus, small cumulus, and itty bitty clouds
  - Configurable in the client config
- Added Distant Horizons support #38 #50 #148
  - Tested for 2.3.2-b
  - Added cloud shadows when using Distant Horizons (toggleable in client config)
  - Cloud render distance uses Distant Horizons render distance. Increase the DH render distance to allow clouds to render farther
- Updated cloud spawning and handling #13 #79 #109
  - The way clouds spawn and move are now handled by the CPU
  - No more static voronoi grid
  - Cloud regions can be spawned/modified using ``/simpleclouds clouds`` commands
  - Currently, clouds can only spawn as (stretched out) circles. More shapes may come soon
  - How each cloud type spawns is completely data driven. Documentation coming soon
- Added a toggleable atmospheric cloud layer
  - How this layer looks depends on the biome
  - Hot/cold biomes have sparse, long strips of clouds
  - Warm biomes like Jungles have blobs
  - Other biomes have long thick streaks
  - Other biomes like oceans have limited atmospheric clouds that appear in large, translucent blobs
- A work-in-progress and early API has been added. Check out https://github.com/nonamecrackers2/simple-clouds-api for more information #1
- Storm fog now has "density" to it instead of appearing solid
- Added some more performance-related config options, mainly the "Generation Interval" option:
  - This option determines how "fast" the cloud mesh should be generated. The mod typically generates the mesh over several frames. This option adjusts how many frames this process takes
  - STATIC: Targets the existing "Frames To Generate Mesh Option"
  - TARGET_FPS: Targets a certain perceived framerate for mesh (default option)
generation/updating
  - DYNAMIC: Adjusts the mesh generation interval depending on your FPS.
Better for lower end PCs
- Total cloud types that can be loaded at once has been increased from 32 to 64 (TSEG) #9
- Migrated to instanced rendering, optimizing memory usage
- Clouds will now slowly fade in when rendered for the first time or when enough time has passed, causing them to "refresh"
- Distance based cloud fog is no longer transparent but instead a solid color
- Shaded clouds have been optimized and is now enabled by default
- The mod now uses periodic simplex noise to prevent issues when out in the world at large coordinate values
- Added a cloud isometric renderer in the cloud previewer screen. To render, press S
- You can now sleep to skip thunderstorms
- Adjusted the horizon and sky color transition when going in/out of a rain/storm cloud
- Reduced the overall storminess of all rain/storm clouds slightly
- Amplified bass of close thunder sounds
- Added the "concurrent compute dispatch" client config option that uses a slightly optimized rendering algorithm at the cost of higher memory usage. Experimental
- Replaced Minecraft's rain sounds
- Removed some unnecessary loading messages
- Cleaned up some code

- Fixed some bugs related to pondering with Create #68

- Fixed the channeling trident enchantment #70

- Added Vivecraft support, fixing related issues #114 #73
- Fixed the reload renderer popup appearing multiple times when changing multiple config options
- Fixed crashes related to "NONE" precipitation type, hopefully fixing #17 #78
